Create a Laravel 10 project for a local business management system named "Aqua Hydrate".

- MySQL
- No API, web app only

AUTHENTICATION
--------------------

--------------------
DATABASE STRUCTURE
--------------------
Create migrations exactly as follows:

1) users table
- id (bigint, primary)
- name (string 100)
- email (string 150, unique)
- password
- role (enum: 'owner')
- timestamps

- id (bigint, primary)
- name (string 150)
- phone (string 20)

- timestamps

- timestamps

4) payments table
- id (bigint, primary)
- customer_id (foreign key customers.id)
- amount (decimal 10,2)
- payment_date (date)
- notes (text, nullable)
- created_by (foreign key users.id)
- timestamps

5) expenses table
- id (bigint, primary)
- expense_type (enum: 'business','personal')
- category (string 100)
- amount (decimal 10,2)
- expense_date (date)
- notes (text, nullable)
- created_by (foreign key users.id)
- timestamps

6) invoices table
- id (bigint, primary)
- sale_id (foreign key sales.id)
- invoice_number (string 50, unique)

- timestamps
